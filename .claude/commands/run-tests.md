description: Führe Tests aus und zeige Coverage-Report
---

Führe Projekt-Tests aus:

1. **Prüfe Test-Setup:**
   - Validiere pytest-Konfiguration
   - Checke ob alle Test-Dependencies installiert sind

2. **Führe Tests aus:**
   - Unit-Tests (schnell, ohne Hardware)
   - Integration-Tests
   - Optional: Hardware-Tests (mit Warnung dass Hardware nötig ist)

3. **Coverage-Report:**
   - Zeige Test-Coverage pro Modul
   - Markiere Module unter 80% Coverage
   - Liste ungetestete Funktionen

4. **Test-Fehler:**
   - Zeige fehlgeschlagene Tests
   - Analysiere Fehlerursachen
   - Schlage Fixes vor

5. **Empfehlungen:**
   - Welche Tests fehlen
   - Wo Coverage verbessert werden sollte

Gib klare Befehle zum Ausführen der Tests.
